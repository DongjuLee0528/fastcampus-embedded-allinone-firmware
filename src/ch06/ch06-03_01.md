# 페리페럴 제어하기 5 - Timer/Counter OVF Interrupt 이론

---

## 1) Timer/Counter란?

- **Timer** : 내부 클럭을 사용하여 일정 시간 간격의 펄스를 만들어 내거나 일정 시간 경과 후에 인터럽트를 발생시키는 기능
- **Counter** : 외부(TOSC1, TOSC2, T1, T2, T3)를 통해서 들어오는 펄스를 계수하여 Event Counter로서 동작되는 것

> 입력소스는 다르지만 사용목적과 결과가 같기 때문에 혼용되어 사용된다.

### 타이머 vs 카운터 비교

| 구분 | 클럭 | 속도 | 분주 | 클럭 사용 방식 | 동작 모드 |
|------|------|------|------|--------------|----------|
| 타이머 | 내부 클럭 | 빠름 | 분주 가능 (범위 내에서 선택) | 선택 가능 | 동기 모드 |
| 카운터 | 외부 클럭 | 느림 | 분주 불가능 | 외부 클럭 그대로 사용 | 비동기 모드 |

### 타이머/카운터 동작 과정

```
내부클럭(동기)  ──┐
                  ├──▶ Prescaler ──▶ Control Logic ──▶ OVF  : 0x00 ~ 0xFF로 카운트 후, 0x00으로 overflow 되면서 인터럽트 발생
외부클럭(비동기)──┘                                   └──▶ COMP : OCR 값과 비교하여 매치되면 인터럽트 발생
```

## 2) Timer/Counter의 활용 - Normal 모드

- **Normal 모드** → 순수 Timer/Counter용으로 파형을 생성하지 않음

### Normal Mode 특징

- 일반적인 타이머 오버플로우 인터럽트가 필요할 때 사용
- **상향카운터 방식** → `0x00 ~ 0xFF` 계수 동작을 반복
- 카운트 도중 Clear 없음
- **OVF 인터럽트** : MAX = `0xFF` 값일 때 발생 (0xFF → 0x00으로 넘어가는 순간)

### 데이터시트 발췌 (14.7.1 Normal Mode)

> The simplest mode of operation is the normal mode (WGM02:0 = 0). In this mode the counting direction is always up (incrementing), and no counter clear is performed. The counter simply overruns when it passes its maximum 8-bit value (TOP = 0xFF) and then restarts from the bottom (0x00). In normal operation the Timer/Counter overflow flag (TOV0) will be set in the same timer clock cycle as the TCNT0 becomes zero.

- TOV0 Flag는 9번째 비트처럼 동작하며, set만 되고 자동으로 clear되지 않음
- 타이머 오버플로우 인터럽트와 결합하면 소프트웨어적으로 타이머 해상도를 높일 수 있음
- Normal 모드에서 파형 생성을 위한 Output Compare 사용은 권장하지 않음 (CPU 점유율 높아짐)

## 3) Interrupt Vector Table

ATmega328P의 인터럽트 벡터 테이블 중 **Timer/Counter0 OVF** 관련 항목:

| Vector No. | Program Address | Source | Interrupt Definition |
|-----------|----------------|--------|---------------------|
| 8 | 0x000E | TIMER2 COMPA | Timer/Counter2 Compare Match A |
| 9 | 0x0010 | TIMER2 COMPB | Timer/Counter2 Compare Match B |
| 10 | 0x0012 | TIMER2 OVF | Timer/Counter2 Overflow |
| 11 | 0x0014 | TIMER1 CAPT | Timer/Counter1 Capture Event |
| 12 | 0x0016 | TIMER1 COMPA | Timer/Counter1 Compare Match A |
| 13 | 0x0018 | TIMER1 COMPB | Timer/Counter1 Compare Match B |
| 14 | 0x001A | TIMER1 OVF | Timer/Counter1 Overflow |
| 15 | 0x001C | TIMER0 COMPA | Timer/Counter0 Compare Match A |
| 16 | 0x001E | TIMER0 COMPB | Timer/Counter0 Compare Match B |
| **17** | **0x0020** | **TIMER0 OVF** | **Timer/Counter0 Overflow** ✅ |

### avr/interrupt.h 매크로 정의 (주요 항목)

```c
#define TIMER2_COMPA_vect_num  7
#define TIMER2_COMPA_vect      _VECTOR(7)   /* Timer/Counter2 Compare Match A */

#define TIMER2_COMPB_vect_num  8
#define TIMER2_COMPB_vect      _VECTOR(8)   /* Timer/Counter2 Compare Match B */

#define TIMER2_OVF_vect_num    9
#define TIMER2_OVF_vect        _VECTOR(9)   /* Timer/Counter2 Overflow */

#define TIMER1_OVF_vect_num    13
#define TIMER1_OVF_vect        _VECTOR(13)  /* Timer/Counter1 Overflow */

#define TIMER0_COMPA_vect_num  14
#define TIMER0_COMPA_vect      _VECTOR(14)  /* TimerCounter0 Compare Match A */

#define TIMER0_COMPB_vect_num  15
#define TIMER0_COMPB_vect      _VECTOR(15)  /* TimerCounter0 Compare Match B */

#define TIMER0_OVF_vect_num    16
#define TIMER0_OVF_vect        _VECTOR(16)  /* Timer/Counter0 Overflow */  ← 핵심
```

## 내용 정리

| # | 핵심 내용 |
|---|---------|
| 01 | 타이머의 Normal 모드는 0부터 최댓값까지 카운트를 반복하며, **최댓값에서 0으로 돌아가는 순간 오버플로우(OVF)가 발생**한다. |
| 02 | 오버플로우가 발생하면 CPU에 인터럽트 신호가 전달되어, **정해진 주기마다 특정 작업**을 수행하게 할 수 있다. |
| 03 | CPU는 인터럽트 벡터 테이블의 **`TIMERx_OVF`** 항목을 참조하여, 해당 타이머의 오버플로우 ISR 코드를 찾아 실행한다. |
